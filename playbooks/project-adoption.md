# Playbook - 项目落地路线（v0）

## 阶段 0：初始化（1 天）
1. 在 `projects/<project-name>/.harness/` 创建：
   - GOVERNANCE.md
   - BACKLOG.md
   - ROADMAP.md
   - TASKS/
   - RUNS/
   - REVIEWS/
2. 固化质量门（对齐现有 Python 项目）
   - pytest（unit + integration 分层）
   - ruff/flake8（若已有）
   - 覆盖率门槛（当前 README 已描述 75%）

## 阶段 1：试运行（1~2 周）
选择 3 类任务各 1 个做闭环：
- A 类：纯代码修复
- B 类：接口/驱动能力扩展
- C 类：文档与测试补强

每个任务严格执行：任务卡 -> 实施 -> 验证 -> 评审 -> 复盘。

## 阶段 2：自动化增强（第 3~4 周）
- 任务卡字段结构化（便于脚本统计）
- 自动生成周报指标（Lead Time / Rework）
- 引入“失败原因分类码”（需求不清/环境问题/测试缺口/架构冲突）

## 阶段 3：飞轮化（第 2 月起）
- 每周固定 30 分钟 Harness 复盘
- 每月一次治理规则修订（GOVERNANCE）
- 持续收紧高频失败点对应的质量门

---

## 推荐首批任务（robot-controller）
1. 建立 `.harness` 基础结构 + 首版治理文件
2. 补齐 `docs/` 中“开发者执行手册”（统一本地/CI 命令）
3. 选一个实际功能需求跑通全链路并沉淀模板

## 完成判据
- 至少 3 个任务完整闭环且有证据
- 至少 1 次规则更新来自复盘结论
- 指标开始可追踪（哪怕先手工统计）
